# 프론트엔드 개발 지침

## 프로젝트 개요
- React 19 + Vite 기반 프로젝트
- 모던 JavaScript/JSX 사용

## 코드 작성 원칙

### React 컴포넌트
1. **함수형 컴포넌트 사용**
   - 모든 컴포넌트는 함수형으로 작성
   - React Hooks를 적극 활용 (useState, useEffect, useMemo, useCallback 등)

2. **컴포넌트 구조**
   - 단일 책임 원칙: 하나의 컴포넌트는 하나의 역할만 수행
   - 재사용 가능한 컴포넌트로 분리
   - Props는 명확하고 최소한으로 유지
   - 컴포넌트 파일명은 PascalCase 사용 (예: `UserProfile.jsx`)

3. **상태 관리**
   - 로컬 상태는 useState 사용
   - 복잡한 상태 로직은 useReducer 고려
   - 불필요한 리렌더링 방지를 위해 useMemo, useCallback 활용
   - 상태는 최대한 하위 컴포넌트에 가깝게 배치

### 성능 최적화
1. **렌더링 최적화**
   - 불필요한 리렌더링 방지
   - React.memo()를 필요한 곳에만 사용
   - key prop을 고유하고 안정적인 값으로 설정
   - 큰 리스트는 가상화(virtualization) 고려

2. **번들 최적화**
   - 동적 import를 활용한 코드 스플리팅
   - 이미지 최적화 (적절한 포맷, 크기)
   - 필요한 라이브러리만 import

### 코드 품질
1. **클린 코드**
   - 의미있는 변수명과 함수명 사용 (한글 주석은 OK, 하지만 변수명은 영어)
   - 함수는 짧고 명확하게
   - 매직 넘버 사용 금지 (상수로 정의)
   - 중복 코드 제거

2. **에러 처리**
   - try-catch로 에러 핸들링
   - 사용자 친화적인 에러 메시지
   - 에러 바운더리 활용

3. **타입 안정성**
   - PropTypes 또는 TypeScript 활용 고려
   - null/undefined 체크

### API 통신
1. **Axios 사용**
   - API 함수는 별도 파일로 분리 (`apis/` 디렉토리)
   - 에러 핸들링 철저히
   - 로딩 상태 관리
   - 재사용 가능한 API 함수 작성

2. **비동기 처리**
   - async/await 사용
   - Promise 체이닝 지양
   - 에러 핸들링 필수

### 스타일링
1. **일관성 유지**
   - 프로젝트 전체에서 동일한 스타일링 방식 사용
   - CSS 변수 활용으로 테마 관리
   - 반응형 디자인 고려

2. **네이밍**
   - BEM, CSS Modules 등 일관된 네이밍 컨벤션
   - 명확하고 의미있는 클래스명

### 접근성 (a11y)
1. **시맨틱 HTML**
   - 적절한 HTML 태그 사용 (button, nav, main, article 등)
   - div 남용 지양

2. **ARIA 속성**
   - 필요한 곳에 aria-label, role 등 추가
   - 키보드 네비게이션 지원

### 파일 구조
```
client/
  src/
    components/     # 재사용 컴포넌트
    pages/         # 페이지 컴포넌트
    apis/          # API 호출 함수
    hooks/         # 커스텀 훅
    utils/         # 유틸리티 함수
    assets/        # 이미지, 폰트 등
    styles/        # 전역 스타일
```
