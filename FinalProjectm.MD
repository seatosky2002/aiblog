WebSocket 기반 실시간 가격 모니터링 대시보드

이 프로젝트는 WebSocket 스트림으로 전달되는 고빈도 금융 데이터를 실시간으로 처리하고 시각화하는 대시보드입니다.
주식/코인 가격 변동을 초 단위로 갱신하며, 사용자에게 부드럽고 빠른 실시간 그래프 경험을 제공합니다.

실제 서비스(업비트, 바이낸스, 트레이딩뷰 등)에서 사용하는 방식과 동일한 패턴을 적용하여
프런트엔드 실무에서 요구되는 성능 최적화, 상태 관리, 스트리밍 처리 능력을 모두 경험할 수 있습니다.

1. 실시간 가격 스트리밍(WebSocket)

거래소 WebSocket 엔드포인트로 연결하여 초당 변동하는 가격 데이터 수신

연결 끊김 복구(reconnect) 및 자동 cleanup 처리

고빈도 데이터(초당 1~10회 업데이트)를 끊김 없이 UI에 반영

2. 실시간 차트 업데이트 (Recharts / Chart.js)

가격이 들어올 때마다 라인 차트에 즉시 반영

가격 히스토리 그래프는 “최근 60초 버퍼”만 유지하여 성능 관리

차트 성능 저하 방지를 위한 데이터 누적 관리 알고리즘 구현

3. 데이터 버퍼링(60초 Sliding Window)

최근 60초 가격만 유지하는 슬라이딩 윈도우 구조

초당 데이터가 쌓여도 메모리 과다 사용 방지

WebSocket → 버퍼 → 그래프 사이의 최적 파이프라인 구성

4. 실무형 성능 최적화 기법 적용

useCallback / useMemo 로 불필요한 렌더링 차단

차트 업데이트를 위한 requestAnimationFrame 수준의 최적화 가능

상태 업데이트를 batching하여 성능 향상

WebSocket cleanup → 메모리 누수 방지
